ТЗ: AI-модуль пост-аналитики диалогов из агрегатора сообщений
1) Цель

Сделать AI-модуль, который после закрытия диалога менеджером автоматически:

забирает историю переписки (клиент ↔️ менеджер) из агрегатора;

прогоняет полученный контент через основную модель аналитики;

сохраняет результаты аналитики в БД/админку и (опционально) отправляет событие/уведомление.

2) Событие запуска и входные данные

Триггер: событие dialog_closed (или аналог) в агрегаторе, которое означает, что менеджер завершил диалог.

Вход:

dialog_id (уникальный id диалога)

channel (tg/vk/whatsapp/…)

customer_id (если есть)

manager_id (если есть)

messages[] массив сообщений (минимум: id, timestamp, author_role: customer|manager|system, text, attachments_meta?, source_message_id?)

Важно: модуль должен корректно работать, если входом будет либо:

полный массив messages[], либо

только dialog_id и модуль сам подтягивает историю из агрегатора по API.

3) Общая архитектура пайплайна (двухступенчатая)
3.1. Stage A — “Компрессор” (дешёвая модель)

Задача: привести диалог к компактному, структурированному виду, чтобы снизить токены на Stage B.
